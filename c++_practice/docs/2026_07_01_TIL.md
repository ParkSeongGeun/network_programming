### 프로세서 구조
네트워크 통신을 처리하는 프로그램 단위. 세 가지 역할로 구성
- 프로세서 = 실행중인 프로그램

- **Consumer** — 클라이언트 연결 수신 / 패킷 처리
- **Service** — 비즈니스 로직 판단
- **Supplier** — 외부 서버로 송신 / 응답 수신

---

### process_ vs service_ 구분

```
process_  =  네트워크 연결에서 이벤트 발생 (접속, 패킷 도착, 응답, 끊김)
service_  =  내가 능동적으로 처리 (비즈니스 로직, 자체 타이머)
```

파라미터에 `connection_handler`나 `cid`가 있으면 대부분 `process_`

---

### 주요 함수

| 함수 | 언제 |
|---|---|
| `service_init` / `service_fini` | 프로그램 시작 / 종료 시 (딱 한 번) |
| `process_init` / `process_fini` | 연결 시 / 끊길 때 |
| `process_request` | 패킷 도착 |
| `process_response` | 외부 서버 응답 도착 |
| `service_request` | 내부에서 올라온 요청 처리 |
| `*_timeout` | 주기적 실행 |

---

### 데이터 전달 함수
```
request_protocol()  →  Consumer → Service (내부로 올림)
send_protocol()     →  Service → Supplier (다른 프로세스로 전송)
```

---

### 릴레이 통신
- `svcid 999` = 가는 요청 / `svcid 777` = 오는 응답
- 같은 데이터라도 받는 프로세스 관점에 따라 `process_` ↔ `service_` 바뀜
- **push_routeinfo / pop_routeinfo** — 중계 시 `돌아갈 주소` 저장/복원 (스택 구조)
