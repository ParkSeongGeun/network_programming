## C++ 프로그램 빌드 과정

### 컴파일
컴파일은 c++ 코드를 컴퓨터가 이해할 수 있는 오브젝트 코드로 변경하는 과정  

하나의 소스 파일만 있을 리는 없다.
- 여러 소스 파일을 컴파일하여 만든 오브젝트 파일들을 하나의 실행 파일로 묶는 **링크** 과정이 필요

c++는 컴파일하기 전 전처리 과정을 거침
- 전처리는 소스 파일이 컴파일되기 전에 소스 코드르 변경하거나 확장하는 등의 작업을 의미
- #include, #if, #define과 같이 # 기호로 시작하는 지시문을 해석하고 그에 따라 소스 코드를 변경하여 컴파일러에 전달

1. foo.cpp (소스 코드): 헤더 파일 포함, #define 변경 등 전처리
2. foo`.cpp (소스 코드): 소스 코드를 오브젝트 코드로 변경하는 컴파일
3. foo.o (오브젝트 코드): 라이브러리나 여러 개의 오브젝트 파일을 하나로 연결하는 링크
4. foo (실행 파일)
