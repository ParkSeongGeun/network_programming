### SCP
Secure Copy Protocol로 SSH를 기반으로 파일을 안전하게 원격 복사하는 명령어
```
scp [옵션] [출발지] [목적지]
```

Example
```
# 로컬 -> 원격(업로드)
scp file.txt user@host:/path/to/dest/

# 폴더 통째로 복사
scp -r ./mydir user@host:/dest/

# 특정 포트로 키 파일 써서 업로드
scp -P 2222 -i ~/.ssh/mykey.pem ./report.pdf ubuntu@1.2.3.4:/home/ubuntu/

# 원격 서버 폴더 전체 내려받기
scp -r user@192.168.0.10:/var/log/nginx ./nginx-logs/

```

근데 오늘 


```
scp sNet1stProcd snet1@cgpvrs01:/GST/bin

scp sNet1stProcd gst@cgpvrs01:/GST/bin
```

위에는 비번을 입력, 아래는 하지 않아도 되었음.

왜일까?

- SSH 키 인증 설정 여부의 차이
    - ~/.ssh/authorized_keys에 공개키가 등록 안되고
    - 아예 SSH 키 자체가 없고 패스워드 인증만 설정되었다면 비번을 입력해야됨
    - 내 로컬 머신에 개인키가 있고, 원격 서버의 홈에 공개키가 등록되어 있으면 안해도 됨!

- `cat /home/gst/.ssh/authorized_keys` 에 공개키가 있을 것
