### SCP
Secure Copy Protocol로 SSH를 기반으로 파일을 안전하게 원격 복사하는 명령어
```
scp [옵션] [출발지] [목적지]
```

Example
```
# 로컬 -> 원격(업로드)
scp file.txt user@host:/path/to/dest/

# 폴더 통째로 복사
scp -r ./mydir user@host:/dest/

# 특정 포트로 키 파일 써서 업로드
scp -P 2222 -i ~/.ssh/mykey.pem ./report.pdf ubuntu@1.2.3.4:/home/ubuntu/

# 원격 서버 폴더 전체 내려받기
scp -r user@192.168.0.10:/var/log/nginx ./nginx-logs/

```

근데 오늘 


```
scp sNet1stProcd snet1@cgpvrs01:/GST/bin

scp sNet1stProcd gst@cgpvrs01:/GST/bin
```

위에는 비번을 입력, 아래는 하지 않아도 되었음.

왜일까?

- SSH 키 인증 설정 여부의 차이
    - ~/.ssh/authorized_keys에 공개키가 등록 안되고
    - 아예 SSH 키 자체가 없고 패스워드 인증만 설정되었다면 비번을 입력해야됨
    - 내 로컬 머신에 개인키가 있고, 원격 서버의 홈에 공개키가 등록되어 있으면 안해도 됨!

- `cat /home/gst/.ssh/authorized_keys` 에 공개키가 있을 것

### Include Guard
헤더 파일이 두 번 이상 include될 때 내용을 통째로 스킵하게 하는 문법

```c
#ifndef _SNET_1ST_PROC_H_
#define _SNET_1ST_PROC_H_

// 헤더 내용 (class, #include 등)

#endif
```

- `#ifndef` — 정의 안 됐으면 진입
- `#define` — 값 없이 그냥 "존재함" 표시만
- `#endif` — 블록 끝

---

### C++ 클래스 구조

| 파일 | 역할 |
|---|---|
| `.h` | 클래스 선언 (설계도) |
| `.cpp` | 실제 구현 |
| `main` | 객체 생성해서 사용 |

헤더만 include해도 cpp 구현을 쓸 수 있는 이유는 컴파일러가 헤더 보고 확인 → 링커가 cpp 구현이랑 연결해주기 때문

### `::` 연산자
```cpp
void snet1stProcd::run() { ... }
```
"snet1stProcd 클래스의" 라는 뜻. cpp 파일에서 구현할 때 어느 클래스 소속인지 명시

---

### 생성자
```cpp
snet1stProcd::snet1stProcd(void) : xBus_Protocol_Service()
```
- `snet1stProcd::snet1stProcd` — 생성자. 클래스 이름 = 함수 이름
- `(void)` — 매개변수 없음
- `: xBus_Protocol_Service()` — 부모 클래스 생성자 호출. **생성자에만 붙음**

---

### 상속과 virtual
```cpp
// 부모
virtual int sound();

// 자식
int sound() override;
```
부모 포인터로 자식 객체를 가리켜도 실제 객체 기준으로 함수 호출됨

### Makefile
```bash
make        # 컴파일
make clean  # 결과물 삭제
make all    # 전체 빌드
```
컴파일 성공하면 실행파일 생김. `ls -l` 에서 `*` 붙은 게 실행 가능한 파일
