# 함수, 구조체

---

## 함수 기본 구조

```cpp
int func(int _arg1, int _arg2)
{
    // logic
}
```

- 반환 형식 → 함수 이름 → 매개변수 순서

```cpp
#include <iostream>

int add(int _x, int _y)
{
    return _x + _y;
}

int main()
{
    int add_result = add(2, 3);
    std::cout << "The result of adding 2 and 3 is: " << add_result << std::endl;
    return 0;
}
```

## 매개변수 종류

매개변수는 일반 변수, 포인터, 배열 모두 사용 가능

### 1. 값 전달 (일반 변수)

함수에 값을 넘기면 **복사본**이 전달됨. 원본은 바뀌지 않음.

```cpp
void add_one(int val)
{
    val = val + 1;  // 복사본만 바뀜
}

int x = 10;
add_one(x);
// x는 여전히 10
```

### 2. 포인터 전달

주소를 넘겨서 원본을 직접 수정할 수 있음.

```cpp
#include <iostream>

void change_negative(int *_val)
{
    if (*_val > 0)
    {
        *_val = -(*_val);
    }
}

int main()
{
    int pos = 1;
    std::cout << "The value of pos is: " << pos << std::endl;
    change_negative(&pos);  // &로 주소 전달
    std::cout << "The value of pos after change_negative is: " << pos << std::endl;
    return 0;
}
```

- 변수 앞에 `&` (주소 연산자) 를 붙이면 해당 변수의 주소를 가져옴
- 매개변수가 포인터(`*`)로 선언된 이유는 주솟값을 받기 위해서

### 3. 배열 전달

```cpp
#include <iostream>

int average(int _array[], int _count)
{
    int sum = 0;
    for (int i = 0; i < _count; ++i)
    {
        sum += _array[i];
    }
    return sum / _count;
}

int main()
{
    int arr[] = {1, 2, 3, 4, 5};
    int avg = average(arr, 5);
    std::cout << "The average of the array is: " << avg << std::endl;
    return 0;
}
```

- `int _array[]` 대신 `int *_array`로 써도 동일하게 동작함
- 배열 이름 자체가 첫 번째 원소의 주소이기 때문

### 4. 참조자 전달 (C++ 스타일)

포인터 전달과 결과는 같지만 문법이 훨씬 깔끔함.

```cpp
// 포인터 방식 (C 스타일)
void swap(int* a, int* b)
{
    int temp = *a;
    *a = *b;
    *b = temp;
}
swap(&x, &y);  // 호출 시 & 붙여야 함

// 참조자 방식 (C++ 스타일)
void swap(int& a, int& b)
{
    int temp = a;
    a = b;
    b = temp;
}
swap(x, y);  // 그냥 넘기면 됨, 근데 원본이 바뀜
```

#### 참조자란?

기존 변수에 **별명**을 붙이는 것. 새로운 변수가 아니라 같은 메모리를 공유함.

```cpp
int x = 10;
int& ref = x;  // ref는 x의 별명

ref = 99;
std::cout << x;  // 99 출력 — x가 바뀜
```

- 내부적으로는 컴파일러가 포인터로 처리함
- `cout << ref` 는 컴파일러가 자동으로 `cout << *ref` 로 처리해줌
- `*`를 붙이는 걸 컴파일러가 대신 해주는 것 → **역참조 자동화**

#### 포인터 vs 참조자

| | 포인터 | 참조자 |
|---|---|---|
| 값 접근 | `*` 붙여야 함 | 그냥 변수처럼 사용 |
| 다른 변수로 변경 | 가능 | 불가능 (선언 시 고정) |
| null 가능 여부 | 가능 | 불가능 |
| 초기화 강제 | 아님 | 선언 시 반드시 초기화 |

### 5. const 참조자 전달

구조체나 클래스처럼 **큰 데이터**를 넘길 때 가장 많이 쓰는 방식.

```cpp
// 값 전달 — 복사 발생 → 느림
void print(Person p) { ... }

// 참조자 전달 — 복사 없음, 빠름, 근데 원본 수정 가능 → 위험
void print(Person& p) { ... }

// const 참조자 전달 — 복사 없음, 빠름, 원본 수정 불가 → 베스트
void print(const Person& p) { ... }
```

| | 값 전달 | 참조자 | const 참조자 |
|---|---|---|---|
| 복사 비용 | 있음 | 없음 | 없음 |
| 원본 수정 | 불가 | 가능 | 불가 |
| 주로 쓰는 경우 | int, char 같은 작은 타입 | 원본 수정이 필요할 때 | 큰 타입 읽기만 할 때 |

## 구조체

연관된 데이터를 하나로 묶는 자료형.

```cpp
struct Person
{
    std::string name;
    int age;
    float height;
    float weight;
};

Person adult;
adult.name = "홍길동";  // 멤버 변수는 . 연산자로 접근
adult.age = 30;
```

- 멤버 변수는 메모리에 연속으로 할당됨
- 멤버 접근은 `.` 연산자 사용

### 구조체를 포인터로 다룰 때

```cpp
Person p = {"홍길동", 30};
Person* ptr = &p;

p.name      // 일반 변수는 .
ptr->name   // 포인터는 ->
// ptr->name 은 (*ptr).name 과 완전히 동일
```

### 함수에 구조체 넘기기

```cpp
// 값 전달 — Person 전체가 복사됨
void print(Person p)
{
    std::cout << p.name << " " << p.age;
}

// const 참조자 — 주소만 넘어감, 수정도 막힘
void print(const Person& p)
{
    std::cout << p.name << " " << p.age;
    // p.age = 99;  ← 컴파일 에러
}

int main()
{
    Person p = {"홍길동", 30};
    print(p);
}
```

구조체를 함수에 넘길 땐 거의 항상 `const&`로 넘기는 게 관례.

const가 붙으면 해당 값은 변경할 수 없다.

### 레퍼런스 변수
C++에서 제공하는 3가지 변수 형식
- 일반 변수: 값을 저장하는 변수
- 포인터 변수: 메모리 주소를 저장하는 변수
- 레퍼런스: 변수에 또 다른 이름, 별칭(별명)을 부여

규칙
- 레퍼런스 변수는 선언 후 반드시 참조할 원본 변수를 지정해야함
- 참조할 ㅐㄷ상이 지정된 레퍼런스 변수는 다른 변수를 참조하도록 변경할 수 없다.
- 레퍼런스 변수는 상수를 참조할 수 없다.
    - 상수는 리터럴인데, 리터럴이 있는 메모리 공간은 읽기만 가능
    - 따라서 해당 영역을 참조하여 값을 변경할 수 없도록 상수를 참조하는 것은 안됨
