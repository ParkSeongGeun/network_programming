### 네임스페이스란?
소속을 지정해주는 역할
```
#include <iostream>

int main() {
    int i, j;

    std::cout << "Enter Num: ";
    std::cin >> i >> j;
    std::cout << i << j << "\n";
    return 0;
}
```

std::는 std라는 네임스페이스에 접근할 때 쓰는 표현
- standard의 줄임말
- C++에서 흔히 사용하는 여러 함수/클래스/객체/유틸리티가 정의된 네임스페이스
    - etc: cout, cin, string 등...

네임스페이스는 내부 식별자(형식, 함수, 변수 등)에 범위를 부여해 여러 라이브러리를 포함할 때 이름이 충돌하는 것을 방지하려고함
- 즉, 잘 구분만 한다면 같은 이름의 함수를 여러번 사용할 수 있음

using namespace std를 통해 std::를 생략할 수도 있지만, 분명 내가 오늘 본 코드에선 그렇지 않았음
- 실무/라이브러리 코드에서 안 쓰는 이유는 이름 충돌 위험 때문
- 같은 이름의 함수(etc: min)이 ACE 또는 std에서도 사용될 수 있음
    - 따라서 관행적으로 안붙이는 것을 권장

참고로 Linux에서의 namespace와는 차이가 있음
- C++ namespace
    - 목적코드: 코드 내 이름 충돌 방지
    - 동작 레벨: 컴파일 타임
    - 대상: 함수, 클래스, 변수 이름
    - 사용 예시: 라이브러리 구분
- Linux namespace
    - 목적코드: 프로세스 격리
    - 동작 레벨: 커널/OS 레벨 (런타임)
    - 대상: PID, 네트워크, 파일시스템, 유저 등
    - 사용 예시: Docker / Container
