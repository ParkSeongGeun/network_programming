### 공유 라이브러리 에러
`error while loading shared libraries: libsEnc.so: cannot open shared object file`
실행파일이 필요로 하는 `.so` 파일을 못 찾아서 나는 에러

**[디버깅 순서]**
1. 라이브러리 파일이 서버에 있는지 찾기 - find / -name "libsEnc.so*"
2. ldd로 의존성 확인
3. ldd에 안나오면 라이브러리 복사 (or 넣어주기)

### ldd
실행파일이 어떤 공유 라이브러리(.so)에 의존하는지 보여주는 명령어
`ldd ./sNet1stProcd`

### link / unlink
`ln -s 원본경로 링크경로 # 심볼릭 링크 생성 (바로가기)`
`unlink 링크경로`
`ls -l /GTS`

### grep
`grep -rn "찾을단어" ./ # 전체 디렉토리에서 검색 (재귀, 줄번호)`
`grep -rn '\<LD' ./ # LD로 시작하는 단어만 (단어 경계)`

### LD_LIBRARY_PATH
프로그램 실행할 때 공유 라이브러리를 어디서 찾을지 알려주는 환경변수
```
echo $LD_LIBRARY_PATH                           # 확인
echo LD_LIBRARY_PATH=/GTS/lib:$LD_LIBRARY_PATH  # 임시 설정 (기존 경로 유지)
echo 'export LD_LIBRARY_PATH=...' >> ~/.profile # 영국 설정
```
